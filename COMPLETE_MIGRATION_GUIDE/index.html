    Complete Migration Guide

    One-Command Migration

    Use the complete_migration.sh script to migrate an Access database to RDS PostgreSQL in a single command.

    Usage

    ./migration_tool/complete_migration.sh <path-to-accdb-file> <postgres-url>

    Example: Local PostgreSQL

    ./migration_tool/complete_migration.sh \
  data/csv_export/Clean_Repair.accdb \
  postgresql://postgres:password@localhost:5432/clean_repair

    Example: AWS RDS

    ./migration_tool/complete_migration.sh \
  data/csv_export/Clean_Repair.accdb \
  postgresql://postgres:DoloresFlagstaff9728@database-1.ca3sci02uq0g.us-east-1.rds.amazonaws.com:5432/clean_repair

    What the Script Does

    The script performs a complete end-to-end migration:

    1. Exports Access database to SQLite intermediate format
    2. Uses mdb-tools to export all tables
    3. Creates migration_tool/intermediate.sqlite

    4. Preserves full source data (addresses, phones, emails)

    5. Recreates PostgreSQL schema

    6. Drops all existing tables
    7. Creates schema from migration_tool/create_schema.sql

    8. Includes all tables: sources, customers, work orders, repair orders, items, users, invite tokens, file uploads

    9. Migrates data with intelligent transformations

    10. Converts date formats (handles invalid Access dates like '01/00/00')
    11. Auto-completes old work orders (pre-2020) with invalid completion dates
    12. Converts boolean fields from YES/NO to true/false
    13. Handles orphaned records and data integrity issues

    14. Preserves full source contact information

    15. Verifies migration results

    16. Shows row counts for all tables
    17. Checks for old work orders with NULL datecompleted

    18. Verifies source data quality (phones, emails, addresses)

    19. Provides next steps

    20. Instructions for restoring user accounts
    21. Suggestions for verifying the application

    Prerequisites

    Required Tools

    1. mdb-tools - For exporting Access databases bash brew install mdb-tools

    2. PostgreSQL client - For psql command bash brew install postgresql

    3. Python 3.x with required packages bash pip install -r requirements.txt

    Required Files

    • Access database file (.accdb)
    • PostgreSQL database URL (local or RDS)

    Migration Features

    ✅ Smart Date Handling

    • Detects invalid Access dates (01/00/00, 00/00/00)
    • Auto-completes old work orders (pre-2020) using:
    • clean or treat dates if available
    • datein + 1 day as fallback
    • Auto-completes old repair orders using:
    • returndate if available
    • datein + 7 days as fallback

    ✅ Source Data Preservation

    • Migrates full source records with addresses, phones, emails
    • Adds skeleton records only for missing shipto references
    • Uses append instead of replace to preserve data

    ✅ Data Integrity

    • Removes orphaned work order items
    • Removes orphaned repair order items
    • Validates customer IDs
    • Handles missing foreign key references

    ✅ Complete Schema

    Includes all tables needed for the application: - tblsource - Vendors/sources - tblcustomers - Customer records - tblcustawngs - Inventory/sail items - tblcustworkorderdetail - Work orders - tblorddetcustawngs - Work order items - tblrepairworkorderdetail - Repair orders - tblreporddetcustawngs - Repair order items - user - Application users - invite_tokens - User registration tokens - tblworkorderfiles - File upload records

    Troubleshooting

    Error: "mdb-tools is not installed"

    Install mdb-tools:

    brew install mdb-tools

    Error: "psql is not installed"

    Install PostgreSQL client:

    brew install postgresql

    Error: "Access database file not found"

    Check the path to your .accdb file:

    ls -la data/csv_export/Clean_Repair.accdb

    Error: "Failed to create PostgreSQL schema"

    Check your database URL and credentials:

    psql "postgresql://user:pass@host:port/dbname" -c "SELECT 1;"

    Migration completes but data is missing

    Check the logs during Step 3 for warnings about: - Orphaned records - Invalid dates - Missing foreign keys

    Old work orders still showing in queue

    After migration, check:

    psql "$DATABASE_URL" -c "
SELECT COUNT(*)
FROM tblcustworkorderdetail
WHERE datecompleted IS NULL
  AND datein < '2020-01-01';"

    Should return 0. If not, the date auto-completion logic may need adjustment.

    Source records missing phone/email/address

    Verify the Access database has this data:

    mdb-export data/Clean_Repair.accdb tblSource | head -5

    If the Access database has the data but PostgreSQL doesn't, check the migration logs for errors during source migration.

    Manual Migration Steps

    If you prefer to run each step manually:

    # Step 1: Export from Access
python migration_tool/run_migration.py step1 --access-file data/Clean_Repair.accdb

# Step 2: Create schema
export DATABASE_URL="postgresql://..."
psql "$DATABASE_URL" -f migration_tool/create_schema.sql

# Step 3: Migrate data
export AWS_S3_BUCKET="awning-cleaning-data"
export AWS_ACCESS_KEY_ID="dummy"
export AWS_SECRET_ACCESS_KEY="dummy"
python migration_tool/run_migration.py step3

# Step 4: Restore users (if needed)
python migration_tool/restore_users.py migration_tool/users_backup_YYYYMMDD_HHMMSS.json
